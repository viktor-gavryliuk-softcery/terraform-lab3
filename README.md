# Terraform Lab 3

Короткий опис проєкту: інфраструктура як код (IaC) за допомогою Terraform для лабораторної роботи з хмарних технологій. Репозиторій містить конфігурацію Terraform та інструкції для розгортання та знищення ресурсів.

## Передумови (Prerequisites)

- [Terraform](https://www.terraform.io/downloads) версії 1.x (перевірка: `terraform version`)
- Обліковий запис у хмарному провайдері та налаштовані облікові дані (наприклад, змінні середовища або профіль CLI), якщо конфігурація звертається до реального API
- [Git](https://git-scm.com/) для клонування та синхронізації репозиторію

## Використання Terraform

У каталозі з файлами `*.tf` виконуйте команди в такому порядку.

### `terraform init`

Ініціалізує робочий каталог: завантажує провайдери, налаштовує бекенд для стану (якщо вказано).

```bash
terraform init
```

Запускайте після клонування репозиторію або зміни провайдерів/модулів.

### `terraform apply`

Показує план змін і застосовує його: створює або оновлює інфраструктуру відповідно до конфігурації.

```bash
terraform apply
```

Для неінтерактивного режиму з автоматичним підтвердженням:

```bash
terraform apply -auto-approve
```

### `terraform destroy`

Видаляє ресурси, керовані поточною конфігурацією та станом (зворотна операція до `apply`).

```bash
terraform destroy
```

Або без запиту підтвердження:

```bash
terraform destroy -auto-approve
```

**Примітка:** не комітьте файли з секретами. Використовуйте `*.tfvars` локально або змінні середовища; ці шаблони вже в `.gitignore`, де це доречно.

## Підключення до GitHub

Після створення публічного репозиторію на GitHub додайте віддалений репозиторій (замініть `<ваш-профіль>` та `<ваш-репозиторій>`):

```bash
git remote add origin https://github.com/<ваш-профіль>/<ваш-репозиторій>.git
git branch -M main
git push -u origin main
```

Якщо `origin` уже налаштовано інструментом GitHub CLI, достатньо `git push -u origin main`.
